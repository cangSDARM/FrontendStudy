(Identity, Access Management, 身份、访问管理)。
严格按照 IAM，则访问控制与身份管理平级。
但在现代实践中，凭证管理(Certificate)也被从身份管理中提升到同一层级，形成“身份 + 凭证 + 访问”三支柱结构，也就是 ICAM

包含

- 身份管理
  - 管理身份的全生命周期：创建、验证、激活、变更、停用、删除
  - 核心：身份注册、身份核验、身份生命周期、身份治理
- 凭证管理
  - 管理凭证的全生命周期：颁发、分发、存储、验证、吊销、审计
  - 核心：强认证、凭证安全、凭证生命周期、MFA 管理
- 访问管理
  - 管理权限与访问控制：谁能访问什么、能做什么、何时访问、在何处访问
  - 核心：策略定义、权限授予、访问执行、权限审计、最小权限

例子

- [Keycloak](https://keycloak.org)
- [janssen](https://docs.jans.io/stable/)
- [FreeIPA](https://www.freeipa.org/page/Main_Page)

## 身份管理

### 认证

用户认证一版般有四种方法，可单独或组合使用：

1. 个人信息：口令、PIN
2. 个人物品：智能卡、U盾
3. 生物特征(静态)：指纹、虹膜。由于泄露无法更换，在验证时需要同时验证设备
4. 行为特征(动态)：语音识别、笔迹

认证方法风险评估

- 置信等级
- 潜在影响
- 风险范围

## 凭证管理

### 口令的基本存储

以加盐的散列存储在用户分离的文件中。加盐目的包括

- 防止不同用户的相同口令碰撞
- 增加暴力字典攻击的难度
- 使得攻击者无法猜测该口令是否被用于其他系统

## 访问管理

一般需要以下三个要素

- 资源管理：为控制的资源制定规则。包括凭证要求和访问时的特定属性、环境条件
- 特权管理：建立和维护身份的资格或特权属性
- 策略管理：给定身份和属性、资源、环境条件，策略规定用户身份可以对该资源实施哪些操作

## 信任框架

Trust Framework，为 ICAM 或其他框架中的身份颁发、凭证认证等环节提供一套公开的、可被审计的规则和操作标准

例子

- OITF(Open Identity Trust Framework)：由 OIX(Open Identity Exchange)运营的标准实例，已关闭
- FICAM：美国联邦的 ICAM 实例
- eIDAS：欧洲的 ICAM 实例

## 身份联合

Identity Federation，解决“技术实现”问题：让一个组织信任在另一个组织创建和发布的身份


协议标准

- OAuth：解决授权问题，仅提供权限并不认证
- SAML2.0：解决认证问题，仅登录、证明身份
- OpenID：解决认证问题，仅登录、证明身份
- OIDC(OpenID Connect)：合并 OAuth + OpenID，提供认证+授权

### SSO

Single sign-on, 单点登录。
可让用户通过一次性用户身份验证登录多个应用程序和网站，指代用户体验。
Federation 实现了 SSO，但 SSO 不一定都用 Federation

流程

- 当用户登录应用程序时，应用程序会生成 SSO 令牌并向 SSO 服务发送身份验证请求。
- 该服务会检查用户之前是否在系统中进行了身份验证。如果是，它会向应用程序发送一个身份验证确认响应，以授予用户访问权限。
- 如果用户没有经过验证的凭证，SSO 服务会将用户重定向到中央登录系统并提示用户提交其用户名和密码。
- 提交后，服务会验证用户凭证并将肯定响应发送到应用程序。
- 否则，用户会收到错误消息并且必须重新输入凭证。多次尝试登录失败可能会导致服务阻止用户在固定的时间段内进行更多尝试。

Kerberos 是一种基于票证的身份验证系统，可让两方或多方在网络上相互验证其身份。
