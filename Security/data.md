## 数据库安全

独特需求

- 数据库管理系统(DBMS)复杂程度高，新特性和服务导致漏洞不断出现
- SQL 的独特和复杂
- 经常缺少专职的数据库安全人员
- 多种平台和环境
- 云和本地混合的趋势

### sql 注入

攻击者利用 sql 语法的动态性，注入攻击代码

主要攻击途径

|                                 |       防范       |
| :-----------------------------: | :--------------: |
| 用户输入(键盘、射频、条形码...) |       过滤       |
|  服务器变量(协议头、环境变量)   |       校验       |
|   二阶注入(存储后读取时攻击)    | 读取数据库时校验 |
|     缓存注入(cookie、cache)     |       校验       |

获取数据的不同方式

- 带内攻击(inband)：数据同一条通道回显
- 带外攻击(out-of-band)：数据走别的通道（DNS/HTTP）带出来
- 推理(inferential)：靠数据库关系/输出/报错推导内容

### 访问控制

数据库访问控制主要控制 SQL 的 Select/Insert/Update/Delete/Reference

数据库默认是 RBAC1 + DAC(因为允许级联, A->B->C->D)。
数据库中的角色还包含“实例级”角色(管理服务器中所有数据库)概念。
并通过 RLS、视图、安全策略来模拟实现 ABAC

#### 推理控制

除了外部攻击者外，内部角色也可以根据数据库关系推理出数据。因此需要进行推理控制

### 加密

数据库常见加密

- TDE(Transparent Data Encryption, 透明数据加密)
  - 对数据库物理文件(数据文件、binlog、备份)加密
  - 解密在数据库内，查询的数据仍然是明文
- CDE(Column Data Encryption, 字段加密)
  - 对数据库字段加密
  - 加密、解密都在应用程序，数据为密文
- [FPE](/Security/cipher/README.md#FPE)
